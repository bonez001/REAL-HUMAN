📖 Chapter 7: APIs – Talking to the Internet Without Crying

“An API is like a vending machine: you send it a request, it gives you something back—sometimes candy, sometimes sadness.”

🌐 What Is an API?

API stands for: Application Programming Interface.
It may sound like a dystopian committee, but it’s actually just:

A way for your code to talk to other people’s code. Over the internet. Politely.

🎩 Analogy Time:

  • Calling an API is like placing a to-go order at a restaurant.
  • You send a request. The server sends back a wrapped result (often JSON).
  • Mess it up? You get an error burrito.

🧪 Try This: Fetching Cat Facts

import requests

response = requests.get("https://catfact.ninja/fact")
data = response.json()

print("🐱 Did you know?", data["fact"])

Run it again. New cat facts every time.
You’ve just created a code-powered trivia machine.

You. Just. Talked. To. The Internet. You’re unstoppable now.

🧠 What Just Happened?

  • requests.get(url) → makes a polite GET request
  • .json() → turns the response into a dictionary
  • data["fact"] → grabs the part you want

You’re not browsing the web now.
You’re commanding it.

🧛‍♂️ API Responses Are Usually JSON

{
  "fact": "Cats sleep 70% of their lives.",
  "length": 33
}

That’s a dictionary. A string-based treasure chest.
JSON = the universal language of APIs.

⚠️ API Errors – The Awkward Silence

if response.status_code != 200:
    print("Something went wrong:", response.status_code)

Add error-catching:

try:
    data = response.json()
except ValueError:
    print("API sent back nonsense.")
