📖 Chapter 6: Debugging – Confess Your Sins to the Console

“To err is human. To debug is divine. To blame the AI is...inevitable.”

🧟 What Is a Bug, Really?

A bug is not:

  • A moral failure
  • A sign you should become a Pokemon Trainer

A bug is simply this:

Your code is doing something. But not what you meant.

Maybe:

  • You mistyped a variable name
  • You forgot that computers are overly literal
  • You tried to divide by zero (again)

The good news? Every bug is a clue. A breadcrumb. A puzzle.
And debugging is the art of turning “WHY???” into “ahhhh.”

🧪 Cursor in Action: The Built-In Therapist

Let’s break something on purpose:

def cheer(name):
    print("Go " + Name + "! You got this!")

cheer("Luna")

See the problem?

  • Cursor will underline the mistake
  • It may suggest changing Name to name
  • It might offer life advice too if you ask nicely
AI Tip: Right-click → “Explain this code”
Or comment:
# Why is this error happening?
Cursor will try to explain in words, not tears.

🔍 Strategy 1: The Holy Trinity of Debugging

  1. Read the Error Message
    • It’s usually right. Or almost right.
    • Look for line numbers and keywords like TypeError, NameError, or ExistentialCrisisError (not real...yet)
  2. Use Print Statements Like Confetti 
    def divide(x, y):
    print("x:", x)
    print("y:", y)
    return x / y
  3. Isolate the Problem
    • Comment stuff out
    • Break big blocks into functions
    • Ask: “When did this code go from cute to cursed?”

🧰 Strategy 2: Use Cursor Like a Debugging Butler

Try asking Cursor:

  • "Explain this error"
  • "Fix this function"
  • "Why is this variable undefined?"
  • "Add logging to this function to help me debug"

It’ll:

  • Point to likely issues
  • Rewrite confusing bits
  • Sometimes hallucinate—but helpfully

Remember: Cursor is not infallible, but it’s a great second brain when yours is fried.

✨ Advanced Debugging Moves

try/except: The "Don’t Panic" Button

try:
    result = 10 / 0
except ZeroDivisionError:
    print("Math exploded! Try again.")

This lets your code:

  • Attempt something risky
  • Catch the error
  • Respond with grace (or sass)

Ask Cursor:

# Wrap this in try/except and print a helpful message

Logging > Printing (Eventually)

Once your projects grow, use logging for pro-style bug hunts.

# Replace print statements with logging

Suddenly, your program is writing its own diary.

😌 Emotional Debugging Tips

  • Walk away. Seriously. Go eat something.
  • Rubber duck method: Explain your code to an inanimate object. Cursor is your digital duck.
  • Ask for help. Cursor. Stack Overflow. Your dog. They’re all valid.
  • Celebrate each fix. Each bug is a lesson. And an excuse for a snack.

🧃 Debug Recap Smoothie

  • Print like your life depends on it
  • Read the errors; don’t fear them
  • Use Cursor to refactor, explain, or guess
  • Bugs ≠ failure. They’re feedback. With sharp teeth.
