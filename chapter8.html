🧠 Why Save Data at All?

Imagine this:

  • You build a chatbot that gives amazing life advice.
  • You want to remember every weird question someone asks it.
  • But each time the program runs, it forgets everything like a goldfish in a thunderstorm.

Solution? Persistence.
Your code needs a memory vault. That’s where files and JSON come in.

💾 File Basics – Saving Text Like a Digital Diary

with open("thoughts.txt", "w") as file:
    file.write("Today I taught a toaster to love.\n")
  • open(...) opens a file
  • "w" means "write mode" (overwrites!)
  • file.write(...) saves text
  • with ensures the file is closed safely

Now open thoughts.txt on your machine. It’s real.
You just wrote to disk, baby. ✍️💽

📖 Reading Files – Memory Retrieval for Robots

with open("thoughts.txt", "r") as file:
    contents = file.read()
    print(contents)

Now your code can:

  • Store logs
  • Save progress
  • Hide secret messages in definitely_not_a_diary.txt

🧳 Enter JSON – Your Data in a Tidy Little Suitcase

JSON (JavaScript Object Notation) is:

  • Easy to read
  • Easy to write
  • Used by basically every API
import json

data = {
    "name": "Luna",
    "mood": "curious",
    "snack_log": ["pretzels", "gum", "data packets"]
}

with open("log.json", "w") as file:
    json.dump(data, file)

To read it back:

with open("log.json", "r") as file:
    data = json.load(file)
    print(data["name"])

It’s like giving your code a fridge magnet memory.
